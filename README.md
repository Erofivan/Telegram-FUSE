# Telegram-FUSE (`tgfs`)

`tgfs` монтирует Telegram-чат/канал как файловую систему через FUSE.

Каждый файл хранится как отдельное сообщение с `document` в Telegram.  
Папки реализованы через теги в caption (например `#docs`) и внутренние sentinel-файлы.

## Что умеет

- читать и записывать файлы;
- создавать и удалять файлы;
- переименовывать файлы;
- создавать и удалять директории;
- показывать директории по тегам Telegram-caption.

## Как это работает

- Файлы отправляются через Bot API `sendDocument`.
- Для чтения используется `getFile` + скачивание по file URL.
- Метаданные (индекс файлов) хранятся локально в `/tmp/tgfs_meta_<chat_id>.json`.
- На старте список файлов берется из локального мета-файла, а не из истории чата.

## Требования

- C++17-компилятор (`g++`/`clang++`);
- `fuse3` (headers + library);
- `libcurl`;
- `cmake` (если собираете через CMake);
- `pkg-config` (желательно, для автопоиска `fuse3`).

## Сборка

### Вариант 1: CMake

```bash
cmake -S . -B build
cmake --build build -j
```

Бинарник: `build/tgfs`

### Вариант 2: Makefile

```bash
make
```

Бинарник: `./tgfs`

## Подготовка Telegram

1. Создайте бота через [@BotFather](https://t.me/BotFather) и получите `BOT_TOKEN`.
2. Добавьте бота в чат/канал, который будет использоваться как хранилище.
3. Дайте боту права на отправку файлов (и удаление сообщений, если нужно удаление из ФС).
4. Получите `CHAT_ID` этого чата/канала.

## Запуск

Синтаксис:

```bash
./tgfs <bot_token> <chat_id> <mountpoint> [fuse_options]
```

Пример:

```bash
mkdir -p /tmp/tgmount
./tgfs "$BOT_TOKEN" "$CHAT_ID" /tmp/tgmount -f
```

Размонтирование:

```bash
fusermount3 -u /tmp/tgmount
```

## Структура файлов

- Файл в корне ФС -> файл в Telegram без тега.
- Файл в `/tag/file.txt` -> сообщение с caption вида:
  - `#tag`
  - `TGFS_FILE:file.txt`

Имена папок в ФС соответствуют тегам в Telegram-caption.

## Ограничения и особенности

- Индекс файлов локальный (`/tmp/tgfs_meta_<chat_id>.json`):
  - если файл удалить/добавить в Telegram вне `tgfs`, индекс не обновится автоматически;
  - при удалении мета-файла текущий индекс будет потерян.
- Операции завязаны на Bot API и сеть (задержка/ошибки Telegram влияют на ФС).
- Удаление старых сообщений в Telegram может быть ограничено политиками платформы.
- Симлинки и hardlink не поддерживаются.
- ФС запускается в single-thread режиме (`-s` добавляется в аргументы FUSE).

## Отладка

- Запускайте с `-f` (foreground), чтобы видеть логи в терминале.
- При проблемах проверьте:
  - валидность `BOT_TOKEN`;
  - корректность `CHAT_ID`;
  - доступ бота к чату/каналу;
  - наличие `fuse3` в системе.
